Necessário desenvolver:

1. Configurar Action Mailer com SMTP (ou mailtrap para desenvolvimento)
2. Criar mailer CertificateMailer com métodos:
   - certificate_email(certificate): envia certificado em anexo para o aluno
   - bcc_email(certificate): envia cópia oculta para o instrutor
3. Configurar Sidekiq + Redis para jobs em background
4. Criar job CertificateEmailJob que:
   - Gera o PDF do certificado
   - Envia email para o aluno com PDF anexado
   - Envia BCC para o instrutor
   - Registra log em EmailLog (status: sent/failed, horário, erro se houver)
5. Na interface web, ao clicar "Enviar Todos" no evento, enfileirar jobs para todos os certificados pendentes
6. Criar página de validação offline:
   - Rota GET /validate que aceita parâmetro ?data= (QR Code scanneado)
   - Decodifica o JSON do QR Code
   - Mostra uma página HTML bonita confirmando se o certificado é válido
   - Deve funcionar SEM internet (após carregar a página uma vez) — usar Service Worker ou simplesmente exibir os dados do QR Code que já contém tudo
7. Adicionar à view show do evento: botão "Enviar Todos os Certificados" que dispara os jobs

ESTRUTURA EXISTENTE:
- app/models/certificate.rb (tem uuid, qr_code_data, sent_at, sent_status, event, participant)
- app/models/email_log.rb (tem certificate, recipient, status, error_message, sent_at)
- app/models/event.rb (tem instructor_email)
- app/controllers/api/v1/certificates_controller.rb (já tem resend_email)
- app/controllers/certificates_controller.rb (web, já tem resend_email e download)

IMPORTANTE:
- Sidekiq deve usar Redis local
- O email deve ter assunto personalizado: "Seu certificado de [nome do evento]"
- O PDF deve ser gerado antes de enviar (reutilizar lógica do download)
- A página de validação deve ser simples, bonita, e funcionar com os dados do QR Code
- Registro completo de logs para auditoria
